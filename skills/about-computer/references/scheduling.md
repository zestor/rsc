# Scheduling and Automation

Computer can run tasks on a schedule and proactively notify the user — turning it from a reactive assistant into a persistent autonomous agent.

## Recurring tasks

Set up tasks that run automatically on a schedule:

- "Monitor competitor pricing daily."
- "Check my inbox for investor replies every hour."
- "Send a weekly sales summary every Monday at 9am."
- "Post a daily market update to Slack."

Recurring tasks persist until explicitly cancelled. Each run can access the full set of Computer's capabilities — search, integrations, code execution, document creation.

## One-time delayed actions

Schedule something to happen later:

- "Send this email at 9am tomorrow."
- "Follow up if no reply in 48 hours."
- "Remind me about this on Friday."

Computer pauses, wakes up at the specified time, and continues from where it left off with full conversation context.

## Push notifications

When a scheduled check finds something noteworthy, Computer sends a push notification with the key details — no need to open the app to see what happened.

- Notification includes enough context to understand the update at a glance.
- If nothing new happened since the last check, Computer stays silent.

## Example use cases

- Daily briefing — pull calendar, CRM, and email data every morning and send a prioritized summary.
- Price monitoring — check competitor pricing hourly, alert when changes are detected.
- Follow-up automation — send an email now, check for a reply in 48 hours, follow up if none.
- Recurring reporting — compile weekly metrics from connected tools and deliver as PDF.
