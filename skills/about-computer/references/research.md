# Research Engine

Computer has native access to Perplexity's search infrastructure. All search types can run in parallel.

## Search types

**Web search** — Current events, prices, facts, documentation, how-to guides. Multiple queries run simultaneously to cover different angles of a topic.

**Academic search** — Scholarly papers with titles, authors, DOIs, publication dates, and citation counts. Useful for literature reviews, finding primary sources, and building evidence-based arguments.

**People search** — Professional profiles with role, company, education, and career history. Useful for prospect research, meeting prep, and networking.

**Image and video search** — Visual content discovery across the open web. Find reference images, stock photos, diagrams, and video content.

**Shopping search** — Product listings with prices, availability, and merchant info. Compare options, track deals, and find specific items across retailers.

**Deep page reading** — Fetches any public webpage and extracts its content, optionally summarizing or answering questions about it. Goes beyond search snippets to read the full source.

## How it differs from typical AI search

- Queries the live web programmatically — faster and more reliable than pixel-based browser screenshots.
- Multiple searches run in parallel, not sequentially.
- Results can flow directly into analysis, documents, or other workflows without copy-pasting.
- Academic search hits scholarly databases directly, not just Google Scholar snippets.

## Example use cases

- Research a company before a meeting — pull financials, recent news, key people, and competitive positioning in parallel.
- Build a literature review — search academic databases, read full papers, extract findings, and compile into a structured report.
- Monitor a topic — search for the latest developments, compare against previous findings, and surface what's new.
- Fact-check claims — cross-reference multiple authoritative sources and note discrepancies.
