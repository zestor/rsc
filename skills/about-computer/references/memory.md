# Persistent Memory

Computer remembers the user across sessions and uses that context to personalize every interaction.

## What it remembers

- **Identity** — Name, role, company, team, manager, colleagues.
- **Preferences** — Communication style, formatting preferences, tools used, workflow habits.
- **Projects** — Active projects, goals, deadlines, key stakeholders.
- **History** — Specific facts and key exchanges from past sessions.
- **Corrections** — When the user says "I prefer X" or "don't do Y," Computer remembers for next time.

## How it works

**Automatic learning** — Computer stores durable facts when they come up naturally in conversation. No explicit "remember this" command needed (though that works too).

**Semantic recall** — Computer can search its memory to retrieve specific facts, preferences, and key exchanges from past sessions. This goes beyond simple keyword matching — it understands context and meaning.

**Personalized output** — Deliverables match the user's known preferences automatically. If the user prefers bullet-point summaries, dislikes certain formatting, or uses specific terminology, Computer adapts without being asked.

## What it doesn't remember

- Ephemeral instructions ("make this shorter" applies only to the current task).
- Sensitive data the user explicitly asks not to store.
- Memory can be disabled entirely if the user prefers.
- Incognito Computer tasks do not read from or write to persistent memory.

## Example use cases

- Start a new session and Computer already knows your role, company, and current projects.
- Ask for a report and it matches the format you preferred last time.
- Reference "that analysis we did last week" and Computer retrieves the relevant details.
- Computer recalls your colleagues' names and roles when prepping for internal meetings.
- Ask for a follow-up analysis on a topic you researched before, and Computer picks up where the previous task left off rather than starting from scratch.
