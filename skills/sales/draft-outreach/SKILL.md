# Draft Outreach

Research a prospect, then draft personalized outreach. Never sends generic messages.

## Execution Flow

1. Identify target (person + company, role + company, or email/profile URL)
2. Research: web search for company and person; enrichment and CRM if connected
3. Select personalization hook
4. Draft message in the appropriate channel format
5. Create email draft (if email connector available) or output text

## Connector Enrichment

| Connector      | What It Adds                                  |
| -------------- | --------------------------------------------- |
| **Enrichment** | Verified email, phone, detailed background    |
| **CRM**        | Prior relationship context, existing contacts |
| **Email**      | Create draft directly in the user's inbox     |

## Hook Priority

Pick the highest-available hook type for personalization:

1. **Trigger event** -- funding, hiring surge, product launch (most timely)
2. **Mutual connection** -- shared contact or warm intro path
3. **Their content** -- post, article, talk they published (shows research)
4. **Company initiative** -- public strategic priority relevant to your product
5. **Role-based pain point** -- common challenge for their title (least personal)

## Email Structure (AIDA)

- **Subject**: personalized, under 50 chars, no spam trigger words
- **Opening**: personal hook proving you did research
- **Interest**: their problem/opportunity in 1-2 sentences
- **Desire**: one proof point from a similar company
- **Action**: single low-friction CTA

## Channel Selection

- Verified email available: email preferred (higher response rate), provide DM backup
- No email: LinkedIn connection request (under 300 chars, no pitch) + follow-up template
- Warm intro possible: suggest mutual connection outreach first

## Message Scenarios

| Scenario                     | Lead With                                                        |
| ---------------------------- | ---------------------------------------------------------------- |
| **Cold**                     | Trigger event or their content + one relevant proof point        |
| **Warm / Mutual connection** | How you know them + why reaching out now                         |
| **Re-engagement**            | New reason to reconnect (their news or yours), no guilt-tripping |
| **Post-event**               | Specific memory from conversation + value-add resource           |

## Style Rules

- Plain text only -- no markdown bold, italics, or headers in emails
- Short paragraphs (2-3 sentences max)
- One value prop, one CTA
- Never open with "I hope this email finds you well" or "I wanted to introduce myself"

## Gotchas

- **Research before writing** -- skipping research produces generic outreach that gets ignored; always complete step 2 before step 4.
- **Subject lines make or break it** -- a generic subject gets deleted; reference something specific to them.
- **One CTA only** -- multiple asks ("schedule a call, check out our blog, and watch this demo") dilute response rates.
- **LinkedIn connection requests have a 300-char limit** -- do not pitch in the request; save it for the follow-up message.
- **Markdown does not render in email clients** -- using `**bold**` or `# headers` will show raw asterisks and hashes.
