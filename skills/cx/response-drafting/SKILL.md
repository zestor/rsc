# Response Drafting

Draft professional customer-facing communications adapted to situation, urgency, and channel.

## Response Structure

```
1. Acknowledgment (1-2 sentences) — what they said/are experiencing
2. Core Message (1-3 paragraphs) — the answer, update, or information
3. Next Steps (1-3 bullets) — what YOU will do, what THEY need to do, when they'll hear next
4. Closing (1 sentence) — warm sign-off, reinforce availability
```

## Length by Channel

- **Chat/IM**: 1-4 sentences
- **Support ticket**: 1-3 short paragraphs
- **Email**: 3-5 paragraphs max
- **Escalation response**: As needed, structured with headers
- **Executive communication**: 2-3 paragraphs, data-driven

## Tone Spectrum

| Situation          | Tone                                  |
| ------------------ | ------------------------------------- |
| Good news / wins   | Enthusiastic, warm, forward-looking   |
| Routine update     | Clear, concise, friendly              |
| Technical response | Accurate, detailed, patient           |
| Delayed delivery   | Honest, apologetic, action-oriented   |
| Bad news           | Direct, empathetic, solution-oriented |
| Issue / outage     | Immediate, transparent, actionable    |
| Escalation         | Composed, ownership-taking, confident |

## Tone by Relationship Stage

- **New customer (0-3 months)**: More formal, extra context, proactively offer help
- **Established (3+ months)**: Warm, collaborative, can reference history, more direct
- **Frustrated / escalated**: Extra empathy, urgency, concrete action plans, shorter feedback loops

## Gotchas

- **Never say "the system" or "the process"** — own it with "we." Deflecting to abstractions destroys trust.
- **Dates, not durations** — "by Friday January 24" not "in a few days." Vague timelines feel evasive.
- **One exclamation mark max per response** — excessive enthusiasm reads as insincere when someone is frustrated.
- **Don't hedge on known facts** — "I believe your account..." sounds uncertain. Check the account and state the fact.
- **Name specific people** — "Sarah from our engineering team will investigate" is more trustworthy than "our team will look into it."
- **Don't copy the outage template for minor bugs** — the urgency format implies severity. Match the format to the actual impact.
