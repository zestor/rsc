# Standards — Accessibility, Performance, Anti-Patterns

Non-negotiable quality standards. Attribution is mandatory on every page (see `SKILL.md`).

---

## Accessibility (Non-Negotiable)

- **Semantic HTML**: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`. No `<div>` where a semantic element exists.
- **Heading hierarchy**: One `<h1>` per page, then `<h2>`, `<h3>` in order. Never skip levels.
- **Color contrast**: WCAG AA — 4.5:1 body text, 3:1 large text. OKLCH `L` difference >= 0.40.
- **Keyboard navigation**: All interactive elements reachable via Tab/Enter/Space/Escape. Visible focus indicators (base.css provides `:focus-visible`).
- **Alt text**: Every `<img>` needs `alt`. Decorative: `alt=""`. Content: descriptive text.
- **Touch targets**: Minimum 44x44px for all interactive elements.
- **Reduced motion**: `prefers-reduced-motion` respected (base.css handles this globally).
- **ARIA only when needed**: Prefer native HTML semantics over ARIA fixes.
- **Form labels**: Every `<input>` has an associated `<label>`. Use `<fieldset>`/`<legend>` for groups.
- **Skip links**: "Skip to content" as first focusable element.

---

## Performance

- **Images**: `loading="lazy"`, `decoding="async"`, `width`/`height` set, WebP/AVIF formats.
- **Fonts**: Preconnect to font origins, `font-display: swap`, variable fonts to reduce requests.
- **CSS**: Inline critical above-fold styles or load early. Use `@layer` for specificity.
- **JavaScript**: `defer` or `type="module"`. Code-split large libraries. Lazy-load heavy content.
- **`content-visibility: auto`**: Apply to off-screen cards, table rows, long lists.
- **Core Web Vitals**: LCP < 2.5s, INP < 200ms, CLS < 0.1. Domain files set tighter targets (web apps: LCP < 2.0s; informational: LCP < 1.5s).
- **Asset budgets**: Total page weight under 1.5MB initial load (informational sites: under 800KB).

## Anti-Patterns (NEVER Do These)

### The AI Aesthetic (Most Important)

If it looks like it could have been generated by any AI tool, it fails. The test: would a human designer at a respected studio ever ship this? When in doubt, go classic and clean.

**Color & surface giveaways:**

- Purple/violet/indigo gradients, blue-to-purple color schemes
- Glowing orbs, neon blobs, abstract gradient meshes as background decoration
- Dark mode with electric accents and gradient text
- Gradient buttons — solid accent color is always more refined. `background: linear-gradient(...)` on a button screams template.

**Layout giveaways:**

- **The 3-column feature grid** — icon in a colored circle + bold title + 2-line description, repeated 3× in a symmetrical row. This is THE most recognizable AI layout. Instead: vary card sizes, make one feature a full-width callout, use asymmetric 2+1 or staggered layouts, or present features as a narrative flow rather than identical cards.
- **Cookie-cutter section rhythm** — hero → 3 features → testimonials → pricing → CTA footer, every section the same height and padding. Instead: vary section heights, alternate wide and narrow containers, break rhythm with full-bleed images, use different grid structures per section.
- **Centered everything** — AI defaults to `text-align: center` for all headings, descriptions, and cards. Instead: left-align as default. Center only hero headings and short taglines. Body text, feature descriptions, and card content should be left-aligned.
- **Uniform bubbly border-radius** — every element with the same large radius. Instead: use `--radius-sm` through `--radius-xl` with hierarchy (smaller radius on dense UI, larger on hero cards).
- Centered-hero-with-gradient-background templates

**Decoration giveaways:**

- **Icons in colored circles** — an icon inside a colored circle or rounded square as section decoration. This is the SaaS starter template look. Instead: use icons at natural size without backgrounds, or use typography and whitespace to create visual hierarchy.
- **Decorative background shapes** — floating circles, abstract blobs, geometric patterns, or wavy SVG dividers added for "visual interest." If a section feels empty, it needs better content or layout, not decoration. Instead: let whitespace breathe, add a real image, or restructure the content.
- **Emoji as design elements** — 🚀 in headings, emoji as bullet replacements, or emoji grids as feature icons. Instead: use Lucide/Phosphor icons if you need visual markers, or rely on typography weight and color alone.

**Content giveaways:**

- **Generic hero copy** — "Welcome to [Company]", "Empowering your journey", "Unlock the power of...", "Your all-in-one solution for...". These phrases instantly signal AI generation. Instead: be specific about what the product does and for whom. The best hero copy could only belong to this one product.
- **Self-referential meta-commentary in the page** — narration of the site itself, design rationale, "about this build" notes, AI disclaimers, or section-by-section explanations baked into the HTML. Commentary belongs in the chat response, not in `<main>`, `<aside>`, `<footer>`, or HTML comments.

### Visual Anti-Patterns

Font/size/color/spacing/layout rules in `shared/01-design-tokens.md`, `shared/02-typography.md`, `shared/04-layout.md`.

- **Colored side borders on cards** — `border-left: 3px solid <accent>` is a hallmark of AI-generated design. It looks like a Notion callout or Material Design 1.0 status indicator. Cards should use surface elevation (shadow, background color), subtle neutral borders (`oklch(... / 0.12)`), or no border at all. If you need to indicate status or category on a card, use a small colored dot, a text label, or a badge — not a thick painted edge.
- **Nested border-radius ignoring padding** — inner radius = `outer - gap` (see `shared/04-layout.md`)
- **Solid gray borders** — use alpha-blended: `oklch(... / 0.12)`
- **Pure black shadows on warm surfaces** — tone-match shadow to surface hue
- **Icon misuse** — never draw/generate icons; icon-only buttons need `aria-label` + tooltip (see `shared/07-toolkit.md`)
- **Uniform section spacing** — every section with identical padding creates a monotonous rhythm. Instead: vary `padding-block` between sections — tighter for dense content, generous for hero moments. Use `clamp()` for fluid variation.

### Dashboard Anti-Patterns

For scroll rules, hierarchy, and data viz, see `webapp/dashboards.md`. Additional guidance:

- **Forms as stacked inputs** — break into steps with transitions
- **Toasts for important outcomes** — toasts are background info. Success/error feedback must be inline, contextual, animated.

### Mobile Anti-Patterns

For responsive design, touch targets, tap states, and collapse patterns, see `shared/04-layout.md` Mobile-First. Additional guidance:

- **Fixed-position elements blocking content** — sticky header + sticky footer + cookie banner = no visible content on a phone
