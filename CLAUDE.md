- Version 2.1 adds first-class automatic markdown web search context: `SearchProvider`, `FunctionSearchProvider`, `HTTPMarkdownSearchProvider`, `SearchRecord`, `ArtifactState.search_results`, prompt rendering, `search.complete` logging, and optional `SEARCH_ENDPOINT` example wiring.
- Firecrawl provider implemented: `FirecrawlSearchProvider` posts to `/v2/search`, requests markdown scrape options, formats Firecrawl JSON into markdown, reads `FIRECRAWL_API_KEY` from environment or `.env`, and never stores API keys in source. Use `.env.example` as template.
- Search providers enforce outbound web request concurrency with `SEARCH_MAX_CONCURRENCY`; default is `2` for the Firecrawl free plan, configurable up to `50`.
- OpenRouter supported as alternate LLM backend via `LLM_PROVIDER=openrouter`, `OPENROUTER_API_KEY`, optional provider routing settings, and `OpenRouterClientAdapter`. Install optional SDK with `/Users/chrisclark/Documents/.venv/bin/python -m pip install -e '.[openrouter]'`.
- OpenRouter default model is `z-ai/glm-5.2`; provider routing is restricted to `fireworks`, `wafer`, `cloudflare`, and `friendli`. Config accepts `cloudfare` as an alias and normalizes it to `cloudflare`.
- OpenRouter config intentionally omits non-mandatory `sort`, `http_referer`, and `categories`; adapter uses dynamic SDK import so the optional dependency does not trigger diagnostics when absent.
- Version 2.2 adds skill-aware routing from recursive `SKILL.md` libraries: `SkillDiscovery`, `HybridSkillRouter`, `CapabilityBroker`, `ReferenceLoader`, `SelectedSkill`, `SkillReadiness`, `ArtifactState.selected_skills`, prompt rendering under `## SELECTED SKILLS`, and `skill.route`/`skill.selected` logs.
- Skill routing supports the code-change-handoff corpus via `SKILL_LIBRARY_PATHS`, ranks top-K skills with embedding cosine plus HashingVectorizer-style lexical cosine, and marks selected skills `ready`, `degraded`, or `blocked` based on inferred capabilities.
- OpenAI provider path now defaults to `gpt-5.5` through `OpenAIResponsesClientAdapter`, mapping internal system prompts to Responses API `developer` input blocks and exposing text verbosity, reasoning effort/summary, store, and include settings.
- Live integration test added at `tests/test_live_pipeline.py`; enable with `RSC_LIVE_TESTS=1` in `.env` or shell, then run `/Users/chrisclark/Documents/.venv/bin/python -m pytest tests/test_live_pipeline.py -m live -q -rs`. It makes real Firecrawl and OpenRouter calls and has passed with live credentials after installing the optional `openrouter` SDK.
- `OpenRouterClientAdapter` now carries a `model` attribute so memory distillation/compression uses the configured OpenRouter model instead of the OpenAI default.
- Detailed structured telemetry added across role calls, evaluator grading, orchestrator flow, artifact updates, and memory lifecycle. Logs use `schema_version=rsc.log.v2`, full text payloads plus SHA-256 hashes/previews/counts, token estimates, success flags, and cumulative state metrics for AI-readable success/failure analysis. API keys and credentials are not logged.
- Daily JSONL file logging is available via `configure_daily_file_logging`; `examples/run_loop.py` enables it using `LOG_DIR` (default `./rsc/logs`) and writes `rsc-YYYY-MM-DD.jsonl`, switching files when the UTC date changes. Generated `rsc/logs/*.jsonl` files are gitignored; `rsc/logs/.gitkeep` preserves the directory.
- FastAPI + React UI added: `rsc.web_api:create_app` exposes `/api/health`, `/api/config`, `/api/runs`, `/api/runs/stream`, and serves the built React app from `rsc/web/static` in the same process. React source lives in `ui/`; build with `cd ui && npm install && npm run build`. Start server with `/Users/chrisclark/Documents/.venv/bin/python -m uvicorn rsc.web_api:app --host 127.0.0.1 --port 8000`.
- Web UI now defaults to OpenRouter `z-ai/glm-5.2` with Firecrawl search and hides manual skill selection. Skill routing is automatic: intent is matched against configured skill libraries before LLM calls and only relevant skill markdown is injected.
- `RSCConfig.from_env()` checks both the current working directory and the RSC project root for `.env`, so `uvicorn rsc.web_api:app` can be launched from outside the project and still load `/Users/chrisclark/Documents/rsc/.env`.
- UI is Perplexity-style with mode/model selectors, attachments, live streamed activity events, and answer panel. Attachments are converted to markdown server-side with `pypdf` for PDF, `python-docx` for DOCX, direct text decoding for markdown/text/YAML/RST, and table/code-block rendering for CSV/JSON.
- `/api/runs/stream` accepts multipart form data (`task`, `mode`, `model`, `rubric_json`, `files`) and returns Server-Sent Events including attachment conversion, captured RSC log events, and final result/error events. React UI reads the stream via `fetch()` and renders the activity timeline live.
- UI polish pass made the app a compact chat surface: transcript and collapsible activity stream above the composer, dense mode/model controls, icon-only submit, attachment chips with file icons and filenames, and no redundant provider/model/search cards.
- Streaming now emits user-facing `activity.update` events in addition to raw logs: request preparation, context decomposition, search query/result status, skill selection, role start/complete summaries, evaluation, and final status. Raw `log.*` records are hidden from the main UI timeline.
- Large attachments are chunked with `chunk_converted_documents`; multi-chunk inputs run sequential section verification before a final synthesis pass, preserving RSC’s sequential loop while avoiding one huge prompt.
- Large-document streaming now persists artifacts under `state/web_runs/{run_id}/`: manifest, per-chunk source/task/result/summary/metadata, all section summaries, and bounded final synthesis task. Final synthesis uses bounded section summaries instead of full chunk outputs to avoid prompt exhaustion.
- Large attachment handling now forces any extractable attachment through the chunk workflow instead of single-pass RSC. Chunk calls use a scoped cached search provider that searches only the original user question, never full attachment/chunk text. Chunk sizes are capped more tightly before section verification.
- Search planning now generates up to 3 concise queries per loop from the user question plus chunk/section keywords; simple questions stay one query. UI stream renders friendly activity cards in the chat transcript, auto-scrolls on new events, and shows Planner output as a task checklist that marks complete when the run finishes.
- Network retry policy added via `rsc.retry`: transient network operations retry up to 5 times with 5/10/15/20/25 second backoff. Wired through role/evaluator/model calls, state-manager LLM and embedding calls, OpenRouter/OpenAI adapters, Firecrawl/HTTP search, skill embedding calls, and frontend fetch requests. Non-retryable auth/bad-request/not-found/validation/content-filter errors are not retried.
- Provider response streaming added for role calls: `RoleAgent` requests streaming completions where supported, aggregates deltas for normal RSC processing, emits `role.delta` structured logs, and the web SSE bridge forwards them as visible `role.delta` events. OpenRouter adapter passes through `stream=True`; non-stream clients fall back automatically.
- LLM role prompts now require GitHub-flavored Markdown output. React UI renders final answers, user text, and streamed event/detail text with `react-markdown` + `remark-gfm`, including headings, lists, tables, links, blockquotes, inline code, and fenced code blocks.
- Long-answer defaults raised: `MAX_INPUT_TOKENS_PER_CALL=200000`, `MAX_OUTPUT_TOKENS_PER_CALL=65536`, and `MAX_TOTAL_TOKENS_PER_SESSION=2000000`. Synthesizer prompt now targets comprehensive long-form answers up to 30k words unless the user explicitly asks for brevity. Planner prompt and task extraction no longer cap the number of task steps.
- Exhaustion scoring fixed: final output is evaluated before exhaustion status is decided; passing outputs can still return `PASSED`, and exhausted outputs retain the evaluator score instead of forcing legacy `0.0` budget verdicts.
